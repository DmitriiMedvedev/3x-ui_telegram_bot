# Dobrinya VPN Bot v17.1 (Stable Multi-Panel Proxy)

## Что нового (v17.1)

Бот полностью переработан для обеспечения максимальной стабильности в современных окружениях (Ubuntu 24.04/26.04, Python 3.14) и идеальной совместимости с VPN-клиентами.

### Ключевые изменения:
1. **Гибридный сервер подписок (Proxy Mode):** `sub_server.py` v17.1 теперь работает как прокси-агрегатор. Он запрашивает нативные подписки напрямую у панелей 3X-UI, объединяет их и отдает клиенту. Это гарантирует 100% совместимость ссылок с любыми VPN-клиентами (v2rayNG, Streisand, Nekobox и др.).
2. **Резервный генератор (Fallback):** Если панели временно недоступны, сервер подписок автоматически переключается в режим генерации ссылок на основе данных из локальной БД, обеспечивая непрерывную работу.
3. **Авто-маппинг ID:** Исправлена проблема с импортом конфигов. Если в JSON экспорте из 3X-UI указан текстовый `tag` вместо числового `id`, бот автоматически найдет реальный ID через API для корректной регистрации пользователей.
4. **Совместимость с Python 3.14:** Все зависимости обновлены для работы с новым Stable C-API (aiogram 3.15+, pydantic 2.10+).
5. **Поддержка API Token:** Авторизация в 3X-UI через Bearer Token для обхода CSRF-защиты (403 Forbidden).

## Быстрый старт

### 1. Настройка окружения (`.env`)

Скопируйте пример окружения и заполните токены.

```bash
cp .env.example .env
```

Внутри `.env`:
```ini
BOT_TOKEN="токен от @BotFather"
CRYPTOBOT_TOKEN="токен от @CryptoBot" (опционально)
SUB_PORT=8080
SUB_BASE_URL="http://ip_вашего_сервера:8080/sub"
```

### 2. Установка

```bash
cd /root/dobrinya_bot/3x-ui_telegram_bot
python3.14 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 3. Запуск через Systemd

Бот настроен на рабочую директорию `/root/dobrinya_bot/3x-ui_telegram_bot/`.

```bash
# Копирование и запуск
cp dobrinya-bot.service /etc/systemd/system/
cp dobrinya-sub.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable --now dobrinya-bot dobrinya-sub

# Проверка логов
journalctl -u dobrinya-bot -f
```

### 4. Настройка в Telegram

1. `/addserver` — добавьте панель (Login/Pass или API Token).
2. `/addinbound` — импортируйте конфиг. Просто вставьте **Export JSON** из меню Inbounds панели 3X-UI.
3. `/sync` — синхронизируйте всех существующих пользователей на новые инбаунды.

## Команды администратора

| Команда                    | Описание                              |
|----------------------------|---------------------------------------|
| `/admin`                   | Панель управления (кнопки статистики) |
| `/addserver`               | Подключить 3X-UI                      |
| `/addinbound`              | Импорт конфига (Inbound JSON)         |
| `/sync`                    | Принудительный пуш юзеров на панели   |
| `/debugtraffic`            | Сверка трафика (БД vs Панели)         |
| `/userinfo ID`             | Профиль пользователя и его Sub URL    |
| `/setbalance ID СУММА`     | Начислить/списать баланс              |
| `/ban ID` / `/unban ID`    | Управление доступом                   |
| `/addpromo КОД РУБ USES`   | Создать промокод (`_` для автокода)   |
| `/broadcast текст`         | Рассылка по всей базе                 |
