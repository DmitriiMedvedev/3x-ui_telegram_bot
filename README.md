# Dobrinya VPN Bot v16.7 (Stable & Multi-Panel)

## Что нового (v16.7)

Бот полностью обновлен для работы в современных окружениях (Ubuntu 24.04/26.04) и поддерживает новейшие функции 3X-UI панелей.

### Ключевые изменения:
1. **Совместимость с Python 3.14:** Манифест зависимостей актуализирован для работы с новым Stable C-API (aiogram 3.15+, pydantic 2.10+). Это исправляет ошибки компиляции `pydantic-core` на новых ОС.
2. **Поддержка API Token:** Добавлена возможность авторизации в панелях 3X-UI через Bearer Token, что решает проблемы с CSRF (403 Forbidden) на современных форках панели (например, MHSanaei).
3. **Улучшенный парсинг Inbound:** Команда `/addinbound` теперь корректно обрабатывает сложные конфигурации, включая транспорт **XHTTP**, gRPC и WebSocket, а также настройки Reality/TLS.
4. **Стабильная база данных:** Исправлены критические ошибки при работе с JSON в SQLite. Использование абсолютных путей гарантирует синхронизацию данных между ботом и сервером подписок.
5. **Динамические подписки:** `sub_server.py` полностью переписан. Теперь он генерирует ссылки на лету, объединяя все доступные протоколы со всех серверов в один base64-конфиг.

## Быстрый старт

### 1. Настройка окружения (`.env`)

Скопируйте пример окружения и заполните токены.

```bash
cp .env.example .env
```

Внутри `.env`:
```ini
BOT_TOKEN="токен от @BotFather"
CRYPTOBOT_TOKEN="токен от @CryptoBot" (опционально)
SUB_PORT=8080
SUB_BASE_URL="http://ip_вашего_сервера:8080/sub"
```

### 2. Установка

```bash
cd /root/dobrinya_bot/3x-ui_telegram_bot
python3.14 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 3. Запуск через Systemd

Бот поставляется с готовыми unit-файлами, настроенными на рабочую директорию `/root/dobrinya_bot/3x-ui_telegram_bot/`.

```bash
# Копирование и запуск
cp dobrinya-bot.service /etc/systemd/system/
cp dobrinya-sub.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable --now dobrinya-bot dobrinya-sub

# Проверка логов
journalctl -u dobrinya-bot -f
```

### 4. Настройка в Telegram

После запуска напишите боту:
1. `/addserver` — добавьте сервер. Можно выбрать авторизацию по **Логину/Паролю** или по **API Token**.
2. `/addinbound` — добавьте конфигурацию (Inbound). Просто скопируйте **Export JSON** из панели 3X-UI и отправьте боту.
3. `/sync` — команда для принудительной синхронизации всех существующих пользователей на новые сервера.

## Команды администратора

| Команда                    | Описание                              |
|----------------------------|---------------------------------------|
| `/admin`                   | Главная панель управления             |
| `/addserver`               | Подключить 3X-UI (Login/Pass или Token)|
| `/addinbound`              | Импорт конфига через JSON Export      |
| `/sync`                    | Синхронизация юзеров на все панели    |
| `/setbalance ID СУММА`     | Изменить баланс пользователя          |
| `/ban ID` / `/unban ID`    | Блокировка доступа                    |
| `/userinfo ID`             | Подробная статистика и ссылки юзера   |
| `/addpromo КОД РУБ USES`   | Создать промокод                      |
| `/billing`                 | Ручной запуск биллинг-цикла           |
| `/broadcast текст`         | Рассылка всем пользователям           |
