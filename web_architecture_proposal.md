# Dobrinya VPN: Web & Telegram Architecture Proposal

В связи с необходимостью поддержки пользователей без доступа к Telegram, архитектура системы должна быть трансформирована из монолитного Telegram-бота в полноценную SaaS-платформу.

## 1. Концепция "API-First" (Централизация)
Главный принцип новой архитектуры: **Единое Ядро (Backend)**.
Бот и Веб-сайт становятся лишь "витринами", которые общаются с одним и тем же Ядром по API.

* **Backend API (FastAPI / Django REST):** Обрабатывает бизнес-логику (биллинг, генерация конфигов, создание пользователей в 3X-UI, работа с балансом).
* **База данных (PostgreSQL):** Единое хранилище для всех пользователей.
* **Telegram Бот (aiogram):** Общается с Backend API, а не напрямую с БД.
* **Web Frontend (React / Vue.js):** Личный кабинет пользователя на сайте, общается с Backend API.

## 2. Умные Подписки (Smart Subscriptions)
Текущий `sub_server.py` будет заменен на "умный" маршрутизатор.

**Как это работает:**
Когда по ссылке `https://vpn.example.com/sub/<sub_id>` происходит переход, сервер проверяет заголовок `User-Agent`:
*   **Если это VPN-клиент** (v2rayNG, Shadowrocket, Xray, NekoBox и т.д.): Сервер отдает стандартную Base64-строку с конфигурациями.
*   **Если это Браузер** (Chrome, Safari, Firefox): Сервер возвращает красивую HTML-страницу (Landing Page/Dashboard) для этого конкретного пользователя. На этой странице он увидит:
    * **Свой баланс (в рублях) и доступный остаток трафика (в ГБ)**.
    * Инструкции по установке VPN-клиентов на разные ОС.
    * Кнопку копирования своей подписки (ту же самую ссылку).

## 3. Система Аккаунтов (Синхронизация Web и TG)
Так как отправка Email на текущем этапе недоступна (закрыты порты, нет домена), аутентификация на сайте будет происходить **только через Telegram**.

*   **Telegram Login Widget:** Пользователь нажимает на сайте "Войти через Telegram" и авторизуется через свой аккаунт TG.
*   **Альтернативный код доступа:** В боте можно получить временный одноразовый код (или ссылку) для быстрого входа на сайт, если виджет Telegram временно не работает.
*   Это позволяет безопасно идентифицировать пользователя на сайте и подтягивать его баланс из единой базы данных.

## 4. Оплата (Биллинг)
*   **Telegram:** Оплата остается через Telegram Stars и CryptoBot.
*   **Web Frontend:** В первой версии сайта функционал пополнения баланса будет отсутствовать. Главная задача сайта на данном этапе — красивая выдача подписок и отображение статуса (баланс/трафик). Если пользователю нужно пополнить счет, он может сделать это через Telegram-бота (ссылка на которого будет на сайте).

## 5. Обновленный Технологический Стек

*   **Инфраструктура:** Docker, Docker Compose
*   **База данных:** PostgreSQL
*   **Кэш & Брокер задач:** Redis
*   **Фоновые задачи:** Celery или Taskiq (асинхронные задачи по обновлению трафика и выдаче конфигов в 3X-UI).
*   **Core Backend & Sub Server:** FastAPI (Python) - идеально подходит для высоконагруженных API, асинхронной работы с 3X-UI и умного роутинга подписок.
*   **Telegram Бот:** aiogram 3.x (в режиме поллинга или вебхуков).
*   **Web Frontend:** React (Next.js) или Vue (Nuxt) для быстрого рендеринга и SEO.

## План перехода (Roadmap)

1.  **Проектирование БД:** Разработка схемы БД в PostgreSQL (включая Telegram ID как основной идентификатор).
2.  **Разработка Core API:** Создание FastAPI-приложения, перенос туда всей логики из `bot.py` и `database.py`.
3.  **Адаптация Бота:** Переписывание бота так, чтобы он не трогал БД напрямую, а дергал методы Core API.
4.  **Разработка Smart Sub Server:** Создание умного обработчика ссылок подписок с проверкой `User-Agent`.
5.  **Создание Web Frontend:** Разработка информационной страницы подписки (баланс, остаток ГБ) с авторизацией через TG.
6.  **Миграция Данных:** Перенос данных из SQLite в PostgreSQL.
